# TinyMOA Documentation

> Quick reference for the TinyMOA ISA and CIM architecture. For project background and motivation, see the [root README](../README.md).

TinyMOA implements `RV32EC` with `Zicsr`, `Zicond`, and `Zcb` to form `RV32EC_Zicsr_Zicond_Zcb` plus custom CIM extensions.

## Documents

- [ISA Reference](./ISA.md) - full encoding tables, extensions, custom instructions, opcode map
- [CIM Architecture](./CIM.md) - crossbar hardware, control signals, pseudo-code operations, examples


## What's In, What's Out

| | Notes |
|-|-------|
| RV32I base | Full |
| E (Embedded) | x0-x15 registers only instead of x0-x31 |
| C (compressed) / Zca | Full Q0/Q1/Q2\* |
| Zcb | `c.lbu`, `c.lhu`, `c.lh`, `c.sb`, `c.sh`, `c.zext.b`, `c.sext.b`, `c.zext.h`, `c.sext.h`, `c.not`, `c.mul` |
| Zicsr | Full |
| Zicond | `czero.eqz`, `czero.nez` |
| MOA custom (32-bit) | `MOA.V*` (opcode `0x0B`), byte-wise ops |
| TinyQV custom (16-bit) | `C.LWTP`, `C.SWTP`, `C.SCXT`, `C.LCXT` |
| F (floating-point) | *Not implemented* - opcodes reserved for future use |
| RV64 / RV128 | *Not implemented* - RV32 only |

\* *We exclude (but reserve) all compressed FP operations since we are not using the RVF extension. However, [they are required if RVF or RVD extensions are used]()*


## Registers

TinyMOA uses **RV32E**: 16 registers (`x0–x15`) instead of the standard 32.
`gp` and `tp` are pseudo-hardcoded because their values are generated by combinational
logic rather than stored in the register file, saving space.

| Register | ABI Name | Value / Role | Saved By |
|----------|----------|--------------|----------|
| `x0` | `zero` | Hardwired 0, writes are ignored | — |
| `x1` | `ra` | Return address | Caller |
| `x2` | `sp` | Stack pointer | Callee |
| `x3` | `gp` | Global pointer (pseudo-hardcoded) `0x1000400` | — |
| `x4` | `tp` | Thread pointer (pseudo-hardcoded) `0x8000000` | — |
| `x5–x7` | `t0–t2` | Temporaries | Caller |
| `x8` | `s0/fp` | Saved register / frame pointer | Callee |
| `x9` | `s1` | Saved register | Callee |
| `x10–x11` | `a0–a1` | Arguments / return values | Caller |
| `x12–x15` | `a2–a5` | Arguments | Caller |

> *Caller*-saved: caller must preserve these across a call if it needs them.
> 
> *Callee*-saved: the called function must restore these before returning.

### Compressed Register Subset

Compressed instructions encode registers in 3 bits, limiting them to `x8–x15`.
Written with an apostrophe: `rd'`, `rs1'`, `rs2'`. Decode as `value + 8`.

| 3-bit encoding | Register | ABI Name |
|----------------|----------|----------|
| `000` | `x8` | `s0/fp` |
| `001` | `x9` | `s1` |
| `010` | `x10` | `a0` |
| `011` | `x11` | `a1` |
| `100` | `x12` | `a2` |
| `101` | `x13` | `a3` |
| `110` | `x14` | `a4` |
| `111` | `x15` | `a5` |

## Instruction Formats

### 32-bit

| Type | Layout | Used For |
|------|--------|----------|
| **R** | `[funct7][rs2][rs1][funct3][rd][opcode]` | Register-register ops (ADD, SUB, AND…) |
| **I** | `[imm[11:0]][rs1][funct3][rd][opcode]` | Immediates, loads, JALR |
| **S** | `[imm[11:5]][rs2][rs1][funct3][imm[4:0]][opcode]` | Stores |
| **B** | `[imm[12\|10:5]][rs2][rs1][funct3][imm[4:1\|11]][opcode]` | Conditional branches |
| **U** | `[imm[31:12]][rd][opcode]` | Large constants (LUI, AUIPC) |
| **J** | `[imm[20\|10:1\|11\|19:12]][rd][opcode]` | Unconditional jumps (JAL) |

### 16-bit Compressed

Compressed instructions use bits `[1:0]` to signal instruction width:
`00/01/10` = 16-bit (Quadrants 0–2), `11` = 32-bit.

| Type | Layout | Used For |
|------|--------|----------|
| **CR** | `[funct4][rd/rs1][rs2][op]` | Register ops (C.ADD, C.MV, C.JR) |
| **CI** | `[funct3][imm][rd/rs1][imm][op]` | Immediate ops (C.ADDI, C.LI, C.LWSP) |
| **CSS** | `[funct3][imm][rs2][op]` | Stack-relative stores (C.SWSP) |
| **CIW** | `[funct3][imm][rd'][op]` | Wide immediate (C.ADDI4SPN) |
| **CL** | `[funct3][imm][rs1'][imm][rd'][op]` | Loads (C.LW) |
| **CS** | `[funct3][imm][rs1'][imm][rs2'][op]` | Stores (C.SW) |
| **CA** | `[funct6][rd'/rs1'][funct2][rs2'][op]` | Arithmetic (C.SUB, C.AND, C.MUL) |
| **CB** | `[funct3][offset][rs1'][offset][op]` | Branches (C.BEQZ, C.BNEZ) |
| **CJ** | `[funct3][jump target][op]` | Jumps (C.J, C.JAL) |
